# Bilingual Documentation Guide

This project supports English and Simplified Chinese documentation.

## Strategy

- English is the default open-source entry point.
- Simplified Chinese lives under `zh-CN/`.
- Major planning and contributor-facing documents should have both versions.
- Vendor-specific technical claims should use the same official sources in both languages.

## File Naming

Use mirrored filenames when possible:

```text
PROJECT_BRIEF.md
zh-CN/PROJECT_BRIEF.md

README.md
zh-CN/README.md
```

For docs under subdirectories, mirror the path:

```text
docs/contracts/sandbox-http-tool-contract.md
zh-CN/docs/contracts/sandbox-http-tool-contract.md
```

## Translation Rules

- Translate meaning, not word-by-word phrasing.
- Keep product names in English unless the vendor has an official Chinese name.
- Keep code identifiers, file paths, API names, and commands unchanged.
- Keep source links aligned between languages.
- If one language is ahead, add a short note at the top of the older version.

## Priority

Translate in this order:

1. `README.md`
2. `PROJECT_BRIEF.md`
3. Shared contracts
4. Compatibility checklist
5. Vendor guides
6. Runnable example READMEs
7. Field notes
